Script 4 PDF Page 1 dataset overview table now surfaces all annotation health metrics as rows: Bad Crops (empty + tiny <5px, with ⚠ when non-zero), Empty Annotation Files, Invalid Annotation Files, Errors Reading Image Files.

Annotation distribution bar chart added. Page 2+ now shows centroid overview plots (`centroid_overview_N.png`) instead of cross-class separability/all-classes overview. `--cross_class` flag removed from Script 4 and master script; replaced with `--save_class_scatter` prompt (per-class UMAP scatter, off by default). `--pdf_quality` arg exposed as CLI option (default 75).
- **Montage optimization (Script 3)** — Reverted PIL canvas approach back to matplotlib layout (same visual output as before). Optimized: images now decoded + resized via PIL before passing numpy array to `imshow` — skips matplotlib's internal decoder, faster on large grids. Layout, DPI (200), labels, colors all unchanged.
- **PDF bar chart placement fix (Script 4)** — Annotation distribution bar chart moved to after Pipeline Configuration table (was incorrectly rendering before it). PDF page 1 order: Dataset Overview → Pipeline Config → Annotation Distribution → Per-Class Clustering Summary.
- **Reuse crops/embeddings prompt (master script)** — Pre-flight check now detects existing embeddings (`.npy` files) in cropped folder. If found: offers "Reuse existing crops + embeddings?" — Yes skips Steps 1 & 2, No deletes folder and reruns from scratch. If folder exists but no embeddings: overwrite/exit prompt as before. Added `shutil` and `pathlib.Path` top-level imports.
- **Windows file lock fix (Script 4)** — Added `img_bar.close()` after reading PIL image dimensions for bar chart — prevents `PermissionError` during `shutil.rmtree` temp cleanup on Windows.
- **Script 1 threading + JPEG output** — Replaced serial crop loop with `ThreadPoolExecutor` (collect-then-merge, no locks). Added `--num_workers` CLI arg (default 4). Output changed from `.png` to `.jpg` (quality 95) for ~1.5× write speedup. Worker count prompted under "Change default parameters?" gate in master script.
- **GPU-accelerated clustering (cuML backend, Script 3)** — Auto-detects cuML at import; falls back to sklearn/umap-learn CPU if cuML unavailable or no CUDA device. GPU path uses euclidean metric on L2-normalized embeddings (cuML has no cosine support) with `eps_euc = sqrt(2 * eps_cos)` conversion. Per-step timing printed (`[time] kNN`, `DBSCAN`, `UMAP`). Works transparently on both Windows (CPU) and WSL2/Linux (GPU).
- **Cross-platform hardening** — Added `encoding='utf-8'` to all `open()` calls in Scripts 1, 2, 3 (Windows codepage safety). Fixed fallback image glob in Script 3 to `f.suffix.lower() in {'.png', '.jpg', '.jpeg'}` (case-insensitive, catches JPEG crops from Script 1). Added `os.chdir(Path(__file__).resolve().parent.parent)` CWD anchor to master script (safe for `exec.bat`, direct run, and WSL2).
- **Stale centroid overview fix (Script 3)** — Deletes all `centroid_overview_*.png` before regeneration. Prevents stale pages from prior runs polluting the PDF when pipeline reruns with `skip_to_clustering=True`.
